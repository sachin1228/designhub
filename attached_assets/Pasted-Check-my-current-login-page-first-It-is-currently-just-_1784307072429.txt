# Approve Flow

When I approve an applicant:

- Change status to Approved.
- Generate a secure invitation token.
- Save the invitation.
- Send a REAL invitation email.

The email should contain a unique link such as:

/signup?token=xxxxxxxx

Only users with a valid invitation should be able to create an account.

Do NOT use dummy email sending.

Integrate a real email solution.

If any credentials are missing, clearly document the required environment variables.

Invitation links must:

- Expire after a configurable period.
- Work only once.

---

# Reject Flow

When I reject an applicant:

- Change status to Rejected.
- Send a REAL rejection email.

The rejection email should be friendly and encouraging.

Explain that they weren't approved this time but are welcome to improve their portfolio and apply again later.

Include a button linking back to the application page.

Applicants should always be able to submit another application later using:

- Same email
- Same LinkedIn
- Same portfolio

Every reapplication should create a new application record.
